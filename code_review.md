## 三、项目亮点

### 1. 安全性设计
- **API Key 安全管理**：API Key 存储在后端配置文件 `uni-config-center/ai-config/index.js` 中，生产环境前端不存储敏感信息
- **加盐密码哈希**：使用 SHA256 + 随机盐值哈希密码，防止彩虹表攻击
- **Token 过期机制**：所有 Token 都有过期时间（默认24小时），过期后自动失效
- **请求频率限制**：基于用户ID实现每分钟30次请求限制，防止 API 滥用
- **输入参数验证**：所有云函数添加了类型校验、长度限制、内容过滤，防止注入攻击
- **结构化日志**：统一使用 `createLogger` 创建结构化日志，便于问题定位和监控

### 2. 代码结构清晰
- 前端页面和后端云函数分离明确，目录结构合理
- **公共工具模块**：`common/utils/index.js` 封装了验证、日志、错误处理、安全等通用功能
- **统一返回格式**：所有云函数统一返回 `{ code, message, data }` 格式
- **AI调用逻辑统一**：`utils/ai.js` 统一封装所有AI调用，无重复代码
- **统一HTTP调用**：所有云函数统一使用 `uniCloud.httpclient` 进行 API 调用

### 3. 工程化水平高
- **GitHub Actions CI/CD**：自动执行语法检查、单元测试、文档检查
- **Jest 单元测试**：70+个测试用例，覆盖率88%+，覆盖核心工具函数
- **结构化日志系统**：支持 info/warn/error/debug 四级日志
- **错误监控系统**：支持错误捕获、统计和告警
- **环境变量配置**：`loadConfig` 函数统一管理环境变量和默认配置

### 4. 健壮性强
- **错误边界处理**：所有前端异步操作包裹 try/catch
- **网络请求重试**：最多2次重试，指数退避策略
- **用户友好的错误提示**：`getUserFriendlyError` 自动转换技术错误为可读提示
- **统一加载状态**：所有页面都有加载动画和状态显示
- **离线模式回退**：云函数调用失败时自动切换到直接API调用，登录失败时自动进入离线模式

### 5. 代码复用性
- **公共工具函数**：hashPassword、verifyPassword、generateToken、validateUsername 等函数统一管理
- **错误处理包装器**：`asyncErrorHandler` 统一处理异步错误
- **验证函数复用**：所有云函数统一使用 validateToken、sanitizeInput 等验证函数
