## 三、代码亮点

### 3.1 安全性

1. **密码加密存储**
   - 使用 SHA256 + 随机盐值 进行密码哈希
   - 每个用户使用独立的 salt，有效防止彩虹表攻击
   - 数据库中不存储明文密码

2. **API Key 安全**
   - 所有第三方 API Key 存储在云端 uni-config-center 配置中心
   - 前端页面不持有任何密钥，避免密钥泄露风险
   - 云函数从配置中心读取配置，不在代码中硬编码

3. **输入验证与 XSS 防护**
   - 所有云函数入口统一进行输入验证
   - 过滤 `< > & " ' \` ;` 等 XSS 特殊字符
   - 移除 `script` 关键词和 `onclick`、`onload` 等事件处理器
   - 对输入长度进行限制，防止超长输入攻击

### 3.2 代码质量

1. **模块化设计**
   - 公共工具函数统一抽取到 `common/utils/index.js`
   - 包含输入验证、密码加密、Token 生成、日志系统等通用功能
   - 各云函数通过 `require` 引入，避免重复代码

2. **统一日志系统**
   - 所有云函数使用统一的结构化日志格式
   - 日志包含：时间戳、云函数名、用户ID、操作、状态、耗时、错误信息
   - 便于问题排查和性能监控

3. **统一响应格式**
   - 所有接口返回统一格式：`{ code, message, data }`
   - 错误响应包含详细的错误信息和错误码
   - 前端可以统一处理返回结果

### 3.3 工程化

1. **CI/CD 流水线**
   - 配置 GitHub Actions 自动化流程
   - 提交代码自动运行单元测试
   - 保障代码质量

2. **单元测试**
   - 使用 Jest 编写单元测试
   - 覆盖核心工具函数（sanitizeInput、hashPassword、generateToken等）
   - 共 24 个测试用例
