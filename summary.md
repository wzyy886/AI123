## 一、项目概述

### 1.1 项目简介

AI助手是一个基于 uni-app x 框架开发的全能型AI智能助手应用，集成了AI问答、文件分析、图片编辑、视频通话等多种功能。项目采用前后端分离架构，前端使用 uni-app x (Vue 3 + UTS)，后端使用 uniCloud 云函数，AI服务对接阿里云通义千问和通义万相 API。

### 1.2 技术栈

| 分类 | 技术 | 版本 |

| 样式 | SCSS | - |
| 后端 | uniCloud 云函数 | - |
| AI服务 | 阿里云通义千问 | qwen-turbo |
| AI服务 | 阿里云通义万相 | wanxiang-v1 |

## 三、技术亮点

1. **API Key安全管理**: API Key存储在后端配置文件中，生产环境前端不暴露任何敏感信息

## 四、遇到的问题与解决方案

### 4.1 问题1：云函数资源耗尽

**问题描述**: 云函数调用时出现"FC invoke failed, resource exhausted"错误

**原因分析**: 云函数依赖的common/utils模块包含jest等开发依赖，导致部署包过大，执行时资源耗尽

**解决方案**:
1. 简化云函数代码，移除外部依赖
2. 在内联实现核心功能（hashPassword、verifyPassword、validateToken等）
3. 添加离线模式回退机制，云函数调用失败时自动切换到直接API调用
4. 登录失败时自动生成本地Token，进入离线模式

### 4.2 问题2：API Key硬编码

**问题描述**: 前端代码中硬编码了阿里云API Key，存在安全风险

**解决方案**:
1. 将API Key移至后端配置文件（uni-config-center/ai-config/index.js）
2. 前端仅调用云函数，不直接调用阿里云API
3. 开发模式下通过环境变量或本地存储配置API Key

### 4.3 问题3：CI/CD配置错误

**问题描述**: GitHub Actions流水线配置错误，导致构建失败

**原因分析**: 
1. JSON配置文件包含注释，导致验证失败
2. GitHub Pages部署配置错误
3. Node.js版本矩阵配置复杂

**解决方案**:
1. 移除JSON配置文件中的注释
2. 简化CI/CD配置，移除GitHub Pages部署
3. 使用单一Node.js版本（20.x）
4. 测试失败不中断流程

### 4.4 问题4：代码重复

**问题描述**: callAI函数在多个页面中重复实现

**解决方案**:
1. 创建utils/ai.js统一封装AI调用逻辑
2. 所有页面统一使用共享函数
3. 减少重复代码，提高可维护性

### 4.5 问题5：缺少输入验证

**问题描述**: 云函数接收用户输入后未进行充分验证，存在注入攻击风险

**解决方案**:
1. 添加输入参数类型校验
2. 设置输入长度限制
3. 过滤特殊字符（移除`<>`）
4. 添加编程语言白名单验证
