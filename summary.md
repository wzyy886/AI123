# AI辅助编程与工程化实训个人总结

## 一、项目概述

本次实训项目是开发一个基于 uni-app x + uniCloud + 阿里云通义千问 API 的 AI 代码助手应用。项目包含 AI 问答、代码生成、代码审查、代码解释、文件分析、AI P图、视频通话等七大核心功能，以及用户系统、历史记录、收藏等辅助功能。整个项目开发过程中，我大量使用了 AI 辅助编程工具（Trae AI）来提高开发效率和代码质量。

## 二、遇到的最大困难及解决过程

### 困难：数据库记录保存失败问题

在项目开发中期，我遇到了一个非常棘手的问题：云函数调用后，数据库中始终没有记录产生。前端明明显示调用成功，但数据表就是空的。这个问题困扰了我整整两天。

#### 排查过程

1. **第一阶段：怀疑前端问题**
   
   一开始我以为是前端调用方式不对，反复检查 `uniCloud.callFunction` 的参数，确认参数都传对了，但数据库就是没数据。

2. **第二阶段：怀疑云函数逻辑**
   
   然后我在云函数里加了 `console.log`，发现云函数确实执行了，也拿到了数据库返回结果。但就是查不到数据。

3. **第三阶段：找到根因**
   
   直到我仔细看 uniCloud 的错误日志，才发现问题所在：
   
   > "提交的字段在云端数据表的schema文件中不存在"

   原来 uniCloud 的数据库有 schema 验证机制，如果上传的字段不在 schema 定义里，就会被静默忽略。我的 `chat_history` 表只定义了几个基础字段，但代码里传了 `type`、`language` 等额外字段，导致整条记录都没存进去。

#### 解决方案

1. 修改每个数据表的 schema.json 文件，补充缺失的字段定义
2. 将每个字段的权限设置为 `true`，允许读写
3. 重新上传所有云函数和 schema

这个问题让我深刻体会到：**遇到问题不能靠猜，一定要看日志，从错误信息入手**。AI 工具在这个过程中也给了我很大帮助，它帮我分析了可能的原因，并指导我如何查看 uniCloud 的日志。

## 三、对 AI 辅助编程的心得体会

### 1. AI 是效率倍增器，但不能代替思考

在项目开发中，AI 帮我节省了大量时间：
- 写云函数的基础框架，AI 几分钟就能搞定
- 写正则表达式，不用再查文档反复试
- 排查 bug 时，AI 能快速给出可能的原因

但 AI 也不是万能的。有好几次，AI 给出的代码看起来很完美，但一跑就报错。比如图片生成的 API 响应格式，AI 给的是 `output.urls`，但实际阿里云返回的是 `output.results[0].url`。如果我不验证就直接用，就会出 bug。

所以我的体会是：**AI 是很好的助手，但最终的判断和验证还是要靠自己**。

### 2. 写好 Prompt 很重要

同样的问题，不同的问法，AI 给出的答案质量差别很大。我总结了几个写 Prompt 的技巧：

- **说清楚背景**：告诉 AI 你用的是什么技术栈、什么环境
- **列清楚要求**：一条一条列出来，不要笼统地说"帮我写个xx"
- **给具体例子**：比如想要什么格式的输出，给一个示例
- **让 AI 解释思路**：不要只要代码，要让 AI 解释为什么这么写

### 3. 代码审查是真的有用

一开始我觉得 AI 代码审查就是走个形式。但真的用了之后，发现 AI 能找出很多我忽略的问题：
- 安全漏洞：比如前端硬编码 API Key，我自己完全没意识到
- 代码重复：10个云函数都在写同样的工具函数，AI 一提醒我才发现
- 边界情况：输入验证不充分、错误处理不完善

特别是安全性方面，AI 给出的建议让我对 Web 安全有了更深入的认识。

## 四、个人在项目中的具体贡献

作为项目的主要开发者，我负责了以下工作：

### 1. 架构设计

- 设计了整体技术选型：uni-app x 前端 + uniCloud 后端 + 阿里云 AI API
- 规划了 12 个云函数的职责划分
- 设计了 5 张数据库表的结构

### 2. 核心功能开发

- **用户系统**：实现了注册、登录、Token 认证、密码加密（SHA256 + salt）
- **AI 对话**：对接通义千问 API，支持上下文对话
- **代码工具**：代码生成、审查、解释三个功能的云函数和前端页面
- **图片处理**：对接万象 AI 实现图片生成和编辑

### 3. 代码质量优化

- 引入结构化日志系统，统一所有云函数的日志格式
- 提取公共工具模块，消除重复代码
- 增强输入验证，提升安全性
- 添加单元测试，覆盖核心工具函数

### 4. 工程化建设

- 配置 GitHub Actions CI/CD 流水线
- 创建 uni-config-center 统一配置中心
- 编写完整的项目文档（README、API文档、Prompt日志等）

## 五、总结与展望

通过这次实训，我不仅完成了一个完整的 AI 应用项目，更重要的是学会了如何高效地使用 AI 工具来辅助开发。

我最深的感受是：**AI 时代的程序员，会写 Prompt 和会写代码同样重要**。知道怎么问问题、怎么引导 AI 给出高质量的答案，往往比自己埋头写代码效率更高。

但同时我也清楚地认识到，AI 只是工具。真正的架构设计、问题排查、质量把控，还是需要人的经验和判断。打好基础，才能更好地利用 AI 这个强大的工具。

未来我会继续学习 AI 辅助编程的最佳实践，探索更多 AI 和软件工程结合的可能性，让自己成为一名 AI 时代的高效开发者。
